# Playtest -- 2026-03-08 10:03

Seed: 7777 | Ticks played: ~279

## The Experience

Started at tick 0, watched the disease grow from 50K infected in Asia and 500 in South America. The early game pacing felt right -- numbers ticked up visibly each tick without being overwhelming. I focused on the research pipeline: identified both diseases, developed Antiviral-A, ran a clinical trial, deployed it to Asia twice, then developed BSA. By tick 279, infections had hit 1M with 614K dead, and diseases had spread to Africa (around tick 87-90) and Europe (around tick 275).

The research pipeline is the core gameplay loop and it works well. Each step (identify, develop, trial, deploy) feels purposeful. Running field and bench research simultaneously is satisfying.

## What Stuck With Me

**NEW BUG: Blank "Confirm Research" panel when pressing Enter on empty project list.** When field research has no available projects, the panel shows "No projects available." with "[Enter] Select [Esc] Back" -- but pressing Enter transitions to a completely empty "Confirm Research" panel. No project name, no costs, no buttons visible. Pressing Enter again does nothing; only Esc exits. This is a UI state machine bug: the ConfirmProject state is entered even when there's no project to confirm. Reproduced reliably at tick 199 by navigating to Field Research (which had no projects left) and pressing Enter.

**All 61 tests pass cleanly.** 48 unit tests + 13 snapshot tests, zero failures, zero warnings. The test suite is solid.

**Q key is a no-op in snapshot mode.** The hotkey bar always shows "[Q] Quit" but in snapshot mode, pressing Q does nothing (the Quit action is only handled in interactive mode's main loop, not in apply_action). Not really a bug, but slightly misleading.

**Panel toggling works.** Pressing T twice opens then closes the threats panel. Same key opens and closes. Intuitive behavior.

**Help panel is accurate but sparse.** Lists all controls correctly. Mentions [P] Policy but doesn't note it's unimplemented. No mention of Enter key for selections. No gameplay guidance (the research pipeline, how medicines work, what the goal is). A new player would know the keys but not the strategy.

**Doses don't deplete.** After deploying Antiviral-A to Asia twice (200K doses total used), the medicine still shows "100.0K doses" available. The save file confirms `doses: 100000.0` never changes. This is by design (doses = batch size per deployment, not inventory), but the UI doesn't make this clear. A player would reasonably think doses are consumable.

## What I Wanted To Do But Couldn't

- **See a notification when disease spreads to a new region.** Strain Alpha appeared in Africa somewhere between tick 85-90, and Europe got infected around tick 275. Both times I only noticed by checking the map. A "Strain Alpha has reached Africa!" alert would add tension.
- **Queue research.** When field research completes, I wanted the next project to auto-start. Having to re-navigate through the menu every time is tedious.
- **See remaining doses or deployment history.** Since doses appear infinite, I wanted some indication of how many total doses I've deployed and their impact.

## Ideas

1. **Fix the blank confirm panel bug.** When the project list is empty, either hide the "[Enter] Select" hint, or make Enter a no-op that stays on the "No projects available" screen. The state machine should not advance to ConfirmProject when there's nothing selected.

2. **Add disease arrival notifications.** When a disease first appears in a new region, show a prominent alert. This is one of the most important events in the game and it currently passes silently.

3. **Clarify dose semantics in the UI.** Either show "100K doses/deployment" to make it clear this is a batch size, or actually track dose inventory if that's the intended design. Current wording ("100.0K doses") implies a finite supply.

4. **Improve the Help panel.** Add a brief overview of the gameplay loop: "Identify threats -> Develop medicines -> Run clinical trials -> Deploy." Mention Enter for selections. Note that Policy is coming soon.
