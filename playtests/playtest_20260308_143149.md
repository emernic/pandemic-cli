# Playtest -- 2026-03-08 14:31

Seed: 7777 (primary), 9999 (secondary) | Ticks played: ~1566 (to defeat) + ~50 on second seed

## The Experience

I played a full game from tick 0 to defeat at tick 1566. The early game was engaging -- identifying unknown pathogens, developing medicines, choosing what to research first. The mid-game fell apart once I had Antiviral-A developed but no medicine for Strain Beta. I watched South America burn for hundreds of ticks while resources piled up with nothing useful to spend them on. The defeat screen was well-produced and informative. The overall arc felt like: interesting decisions early, helpless stagnation mid-game, inevitable defeat.

## What Stuck With Me

**Good:**
- The research pipeline (identify -> develop -> clinical trial -> deploy) creates a satisfying progression. Each step feels meaningful.
- The untested medicine warning is dramatic and well-designed. Deploying an untested medicine and getting "ADVERSE REACTION: 20.0K died" was a genuine gut-punch moment.
- The defeat summary screen is excellent -- per-region breakdowns, percentage of population lost, clean layout.
- Error messages for invalid actions are clear: "Need 10 RP to boost (have 0)", "No infected population in North America".
- Running field + bench research simultaneously feels like a real strategic choice.

**Bad:**
- The "arrow Oceania" rendering artifact on the map when North America is selected looks like a bug (details below).
- No visual indicator of which map region is selected -- the only way to tell is by which card shows extra disease detail text.
- The game starts in RUNNING state in snapshot mode, not PAUSED as CLAUDE.md suggests it should.

## What I Wanted To Do But Couldn't

1. **See deployment results on the map.** After treating 100K infected in Asia, I wanted to see a visual confirmation -- a flash, a number change animation, anything. The numbers update silently.

2. **Queue research.** I kept having to re-open the research panel after each project completed. I wanted to set up "identify Strain Beta, then develop Broad-Spectrum" and let it run.

3. **See a dose counter that reflects usage.** The medicine panel always shows "100.0K doses" regardless of how many times I've deployed. I expected doses to deplete. If each deployment is a fresh batch, the UI should say something like "100.0K doses per deployment" to make that clear.

4. **Get notified when research completes.** Projects finished silently. I had to keep checking the research panel to notice completion. A status bar message like "Research complete: Antiviral-A developed!" would help.

5. **Understand why Antiviral-A can't help South America.** When deploying Antiviral-A to South America (which has Strain Beta), the target screen shows "0 infected" with no explanation. A message like "No Strain Alpha infections in this region" would prevent confusion.

## Ideas

**Untested medicine as a real dilemma:** The adverse reaction mechanic is already great. Lean into it harder. What if deploying untested medicine in a CRIT region gave a bonus to clinical trial speed (you're collecting real-world data)? Now you have a genuine ethical dilemma: deploy untested medicine to a dying region to save some people AND accelerate your trial, but risk killing others. The player who deploys untested medicine to gather data faster should feel clever and guilty at the same time.

**Research notification system:** When a research project completes, the status bar below the header could flash the result for a few ticks: "BREAKTHROUGH: Antiviral-A developed! Open [M] Medicines to deploy." This keeps the player in the loop without forcing them to poll the research panel. The status bar area already shows research progress -- extend it to show completions.

**Region connection visualization:** The `arrow Oceania` text is trying to show cross-region connections, which is a great idea executed poorly. Instead of a text label that overlaps the grid connectors, highlight the connected region's border when a region is selected. Or draw a colored line between them. The connection information matters for understanding disease spread paths -- it just needs better visual treatment.

**Deploy target disambiguation:** When a medicine targets Strain Alpha but the selected region only has Strain Beta, the deploy screen should say something like: "Antiviral-A targets Strain Alpha. This region has no Strain Alpha infections." Currently it silently shows "0 infected" which is technically correct but confusing when the region card on the map shows 2.8M infected.

**Vaccinate-early validation:** The game lets you select "Treat infected" on a region with 0 infected, shows the untested warning, you confirm, and THEN it tells you there's nobody to treat. The "0 infected" check should happen before the untested warning -- don't ask the player to accept risk for an action that will fail anyway.

## New Issues Found

1. **Rendering artifact: region connection label overlaps grid connectors.** When North America is selected, `↔ Oceania` text appears in the connector row between the top and bottom grid rows. The connection hint renders at the bottom of the selected region card, which for top-row regions lands on the inter-row connector line. (src/ui/region_list.rs around line 171)

2. **Untested medicine warning shows before zero-target validation.** When deploying a treatment to a region with 0 infected, the game shows the untested medicine warning first, asks for confirmation, and only then checks if there are actually infected to treat. The zero-target check should come first.

3. **No visual indicator of selected map region.** Arrow keys select regions on the map, but there's no highlight, border change, or color difference on the selected card. The only indicator is that the selected card shows extra disease detail text, which is not discoverable.

4. **Help panel incomplete.** Lists basic controls but doesn't mention Enter (needed for all panel interactions), doesn't explain the research/medicine workflow, and doesn't mention arrow key map navigation.

5. **Snapshot mode starts RUNNING, not PAUSED.** The game begins in RUNNING state. CLAUDE.md says interactive mode "starts paused" but snapshot mode does not. This may be intentional for snapshot mode but is inconsistent.
